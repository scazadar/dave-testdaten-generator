Das Problem

Testen ohne Testdaten ist mühsam

  • DAVe ist das Verkehrszähldaten-Portal der Stadt München.
  • Wer entwickelt, testet oder vorführt, braucht realistische Zähldaten.
  • Diese von Hand zu erstellen ist mühsam, langsam und fehleranfällig — striktes CSV-Format, 96 Viertelstunden-Intervalle, 16 verschiedene Zählarten mit je eigener Feldlogik.

Bisher

CSV-Zeilen von Hand tippen, Format-Fehler suchen, Datei hochladen, hoffen, dass der Import durchläuft.

Die Herangehensweise

Eine schlanke Web-App, die das Format kennt

1 · Konfigurieren

Zählstelle, Zählart (alle 16), Datum, Knotenarm wählen. Felder nach/Strassenseite/Richtung blenden sich je Zählart automatisch ein.

2 · Werte erzeugen

Pro Fahrzeugkategorie: fester Wert, Zufall, realistische Tageskurve (Morgen-/Abendspitze) oder aus.

3 · Direkt an DAVe

CSV-Download oder per Klick direkt ins Backend senden — kein manueller Datei-Import nötig.

Das Kern-Feature

Generator→ POST /zaehlstelle/save→ POST /zaehlung/save…→ Elasticsearch + Postgres
